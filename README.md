# VocalAssist 🔥

Вокальный тренажёр в одном HTML-файле: загружаешь ноты (.mxl / .musicxml) — приложение показывает вокальную линию на piano-roll, играет аккомпанемент и через микрофон проверяет, попадаешь ли ты в ноты.

**Открыть:** https://keynode.github.io/VocalAssist/

## Возможности

- Piano-roll с падающими нотами и подсветкой клавиш, воспроизведение начинается сразу
- Загрузка нот прямо в браузере, без сервера: MusicXML (.mxl, .musicxml) и MIDI (.mid) — вокальная линия и аккомпанемент разделяются автоматически; в настройках можно выбрать одну или несколько визуализируемых MIDI-дорожек
- MIDI-звук — SpessaSynth с полным GeneralUser GS: сохраняются исходные инструменты, velocity, педали, pitch bend, эффекты и барабаны. Пока качественный движок загружается, воспроизведение без задержки запускается через прежний WebAudioFont
- В `⚙ → SoundFont` можно выбрать свой `.sf2`, `.sf3` или `.dls` (например, распакованный Timbres of Heaven). Банк хранится в IndexedDB этого браузера и включается со следующего воспроизведения
- Библиотека песен в localStorage: добавленные песни остаются, темп и тон запоминаются для каждой
- Микрофон: детекция высоты голоса, точка твоей ноты на линии, отклонение в центах, % попаданий
- Навигация по фразам, повтор фразы, легато, транспонирование, минусовка
- Горячие клавиши: Пробел — петь/пауза, ←/→ — фразы, L — повтор, R — сначала

## Песни в репозитории

Файлы .mxl / .musicxml из папки [`songs/`](songs/) автоматически появляются на полке «Песни» на всех устройствах (значок ♪). Чтобы добавить песню с телефона: открой репозиторий на github.com → папка `songs` → «Add file → Upload files». Через минуту после коммита песня появится в приложении (потяни страницу для обновления).

## Заметки

- Микрофон работает только по HTTPS (на GitHub Pages — работает) и лучше всего в наушниках
- На iPhone: «Поделиться → На экран "Домой"» — тогда браузер не очистит сохранённые песни
- Ноты из PDF: распознать в MusicXML через Audiveris (ПК, бесплатно) или PlayScore 2 (iPhone)

## Сборка MIDI-движка

Собранные браузерные файлы уже лежат в `audio/`. После обновления npm-зависимостей пересоберите их командой `npm run build:audio`. Встроенный GeneralUser GS распространяется вместе с его лицензией в [`audio/GeneralUser-GS-LICENSE.txt`](audio/GeneralUser-GS-LICENSE.txt); лицензия SpessaSynth находится в [`audio/SPESSASYNTH-LICENSE.txt`](audio/SPESSASYNTH-LICENSE.txt).
